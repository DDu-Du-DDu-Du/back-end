# 미리알림 생성(Reminder) 구현 플랜

## 1) 목표

- 기존 `Todo.remindAt(단일)` 중심 모델에서 `Reminder(다수)` 중심 모델로 확장한다.
- `POST /api/reminders` API를 추가하여 로그인 사용자가 특정 투두에 다수 미리알림을 생성할 수 있게 한다.
- Reminder 생성 직후 알림 스케줄 이벤트를 발행해 기존 알림 파이프라인과 연동한다.
- 도메인/애플리케이션/인프라/부트스트랩/API 문서/DB 마이그레이션/테스트를 한 번에 정합성 있게 반영한다.

## 2) 신규/변경 패키지 및 클래스

> 아래 경로는 현재 코드베이스 패턴(`planning` bounded context, `application-common` 포트, `planning-application` 서비스, `planning-infra-mysql` 영속화, `bootstrap/planning-api` 컨트롤러)에 맞춘 구현 기준이다.

- `com.ddudu.domain.planning.reminder.aggregate.Reminder`
  - 파일: `domain/planning-domain/src/main/java/com/ddudu/domain/planning/reminder/aggregate/Reminder.java`
  - 역할:
    - 필드: `id`, `userId`, `todoId`, `remindsAt`, `remindedAt`
    - 생성 검증: `userId`, `todoId`, `remindsAt` null 체크
    - 팩토리: `from(userId, todoId, remindsAt, scheduledAt)`
      - `remindsAt`, `scheduledAt` null 체크
      - `remindsAt > scheduledAt` 검증
    - `isReminded()`
    - `getRemindDifference(LocalDateTime todoScheduledAt)`

## 3) 구현 순서 (권장)

1. **DB 선반영**: `V19__add_reminders.sql` 작성 및 제약조건 확정
2. **도메인 구현**: `Reminder` aggregate + 에러코드 추가
3. **포트/DTO 정의**: `application-common`의 in/out port, request/response 작성
4. **영속 어댑터 구현**: `ReminderEntity/Repository/PersistenceAdapter`
5. **유스케이스 구현**: `CreateReminderService` + 이벤트 발행
6. **API 노출**: `ReminderController`, `ReminderControllerDoc` 작성
7. **기존 Todo 미리알림 경로 정리**: 공존/마이그레이션 전략에 맞춘 코드 정리
8. **테스트 작성 및 회귀 확인**
